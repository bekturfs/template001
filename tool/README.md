# scrollcast — сайт → видео с плавным скроллом

Вставляешь ссылку — получаешь вертикальный mp4, где страница плавно
проскролливается с «пружинной» физикой, как палец на телефоне.
Рендер покадровый, на виртуальном времени браузера, поэтому видео
идеально гладкое на любой машине, а CSS-анимации страницы идут
в правильном темпе.

## Установка

Нужны Node.js 18+, ffmpeg и Chromium:

```bash
cd tool
npm install                 # ставит playwright-core
npx playwright install chromium   # если нет своего Chromium
# ffmpeg: brew install ffmpeg / apt install ffmpeg
```

## Примеры

Простейший случай — любая ссылка:

```bash
node render.js https://example.com --out demo.mp4
```

Пригласительный с конвертом-заставкой и музыкой:

```bash
node render.js https://sezim-kyz-uzatuu.toi-invite.workers.dev \
  --tap "#envelope" \
  --stops ".invite,.band,.details,.program,.rsvp,footer" \
  --music music.mp3 --music-start 13 \
  --out sezim.mp4
```

## Опции

| Опция | По умолчанию | Что делает |
|---|---|---|
| `--out` | render.mp4 | имя выходного файла |
| `--width` / `--height` | 390 / 844 | вьюпорт (iPhone) |
| `--scale` | 2 | devicePixelRatio → видео 780×1688 |
| `--fps` | 30 | частота кадров |
| `--tap` | — | CSS-селектор: кликнуть перед скроллом (конверт, заставка) |
| `--wait` | 2.5 | секунды до клика |
| `--settle` | 4 | секунды после клика до начала скролла |
| `--stops` | авто | секции-остановки через запятую; без опции — по экрану за раз |
| `--dwell` | 2.4 | пауза на каждой остановке, сек |
| `--travel` | 1.6 | длительность проезда между остановками, сек |
| `--end-hold` | 1.5 | удержание последнего кадра, сек |
| `--music` | — | аудиофайл; старт совпадает с моментом `--tap` |
| `--music-start` | 0 | с какой секунды трека начинать |
| `--fade-in` / `--fade-out` | 8 / 2.5 | нарастание и затухание звука, сек |
| `--ffmpeg` | ffmpeg | путь к ffmpeg (или переменная `$FFMPEG`) |
| `--chromium` | авто | путь к Chromium (или `$CHROMIUM_PATH`) |
| `--keep-frames` | — | оставить папку с кадрами (для отладки) |

## Как это работает

1. Страница загружается, ждём шрифты и сеть.
2. Виртуальное время браузера ставится на паузу.
3. Для каждого кадра: позиция скролла вычисляется по кривой
   недодемпфированной пружины (лёгкий накат + мягкая остановка),
   виртуальное время продвигается ровно на 1/fps, снимается кадр
   в полном разрешении.
4. ffmpeg собирает кадры в mp4 и подкладывает музыку с фейдами,
   синхронизируя её старт с кликом по заставке.
